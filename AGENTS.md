# AGENTS.md

本文件为在本仓库中工作的 AI 代理提供持续性的工作指引。该仓库是通用技能集合（Agent Skills），每个技能都是独立、可复用的包。

## 仓库目标

- 维护高质量、可移植的技能包，扩展 AI 助手的能力。
- 保持技能结构一致、文档清晰、示例可运行。

## 目录结构约定

- `skills/<skill-name>/SKILL.md`：必需，包含 YAML 前置元数据与技能说明。
- `skills/<skill-name>/scripts/`：可选，脚本或工具入口。
- `skills/<skill-name>/references/`：可选，参考文档与清单。
- `docs/<skill-name>.md`：面向用户的使用说明文档。

## 新增或更新技能的要求

1. **遵循 Agent Skills 规范**：保持 `SKILL.md` 简洁，建议 500 行以内。
2. **保持技能自包含**：避免跨技能强耦合，脚本尽量参数化。
3. **文档同步更新**：新增或更新技能时，必须同步更新：
   - `docs/<skill-name>.md`
   - `README.md` 的技能列表与推荐阅读顺序
4. **路径稳定**：文档中的引用使用相对路径（如 `../skills/...`），保证从仓库根目录可访问。
5. **示例可运行**：命令示例以仓库根目录为默认工作目录，并明确使用 `python3` 等可执行入口。

## 文档风格

- 语言默认使用中文，与现有文档保持一致。
- 建议章节结构：**概述**、**适用场景**、**核心工具**、**使用方法**、**最佳实践**、**参考资源**。
- 末尾添加 **最后更新** 日期。

## 安全与质量

- 避免直接修改用户本地配置文件，优先提供可复现的命令与说明。
- 如果脚本支持预览与执行模式，默认先预览再应用。
- 不修改与当前任务无关的技能或文档。

## 验证与发布

- 新技能或重大改动建议运行验证脚本（如 `skills/skill-creator/scripts/quick_validate.py`）。
- 提交时将技能源码与文档一起提交，确保阅读路径完整。
